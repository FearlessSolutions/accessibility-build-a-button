Exercise 4 - But does it sound like a button?

If you tested the previous exercise with NVDA or VoiceOver, you may have noticed that when you focused the real button the screen reader actually read off that it was a button, but when you focused your custom button that it just read the text content or something.

If someone depends on a screen reader, that's all they're going to hear and they won't understand that what they're focused on is, in fact, supposed to be a button. We'll fix that in this exercise.

Screen readers can actually be told that your custom button is a button. This is done with what's called an ARIA Role. There are many different kinds of roles, such as checkbox, slider, option, and tab. MDN has a good list of others here. The one you'll need to specify in this exercise is button. (Couldn't see that one coming!)

Try adding the attribute role="button" to your custom button, then try focusing it again while using NVDA or VoiceOver. The screen reader should now report that your button is, in fact, a button. You can check you got it right with npm run exercise-4-verify.
